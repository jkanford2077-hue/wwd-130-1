Developer Report

How did you communicate with your client? Was there any issues?
I communicated with the client through instagram dm's and periodic check-ins. When questions arose, I asked for clarification to avoid misunderstandings. There were no major issues, but ensuring clarity early on helped the project move smoothly.

How did you handle publishing your client's site and sharing it with the client?
I published the site using GitHub Pages and verified that all files were correctly placed and accessible. Once the site was live, I shared the link with the client and confirmed that everything displayed correctly on their end.

How was your experience with managing your site with GitHub?
Managing the site with GitHub was efficient. Version control made it easy to track changes, and GitHub Pages provided a simple way to publish the site. The workflow was straightforward once the repository structure was set up properly.

General Experience and Reflections

What was the most challenging part of this project for you? Were you able to overcome it?
The most challenging part was ensuring that the site structure matched both the client’s expectations and getting things exactly as the wireframe said.

What did you enjoy most about this project?
I enjoyed seeing the site come together from the initial plan to the final published version. learning about the guns that i made the site about was interesting

What would you do differently if you could do the project again?
I would establish clearer documentation from the start and create a more detailed checklist for each stage of development. This would streamline the workflow and reduce the need for mid-project adjustments.

Do you feel more confident in working on real-world projects after this experience? Explain.
Yes, this project increased my confidence. It provided hands-on experience with planning, communication, development, and deployment. Completing each step reinforced my ability to manage real-world tasks and deliver a functional final product.
